# 모리스 메를로-퐁티 대시보드 피드백 및 수정 계획 (Feedback & Improvement Plan)

현재 완성된 대시보드(`app.py`) 코드를 분석한 결과, 전반적으로 데이터 캐싱(`@st.cache_data`)과 예외 처리 등을 훌륭하게 수행하여 안정적인 기반을 다졌습니다. 
성공적인 GitHub 및 클라우드(Streamlit Cloud 등) 연동을 축하드립니다! 다만 클라우드 배포와 향후 확장성을 고려할 때 몇 가지 구도적/기능적 수정이 필요합니다.

아래는 현재 구현된 대시보드의 강점과 비판점(개선점), 그리고 이를 어떻게 고도화할지에 대한 단계별 수정 계획입니다.

## 1. 현재 대시보드 피드백 (비판 및 리뷰)

### 👍 강점 (Strengths)
1. **안정성 및 최적화**: `@st.cache_data`를 적절히 사용하여 매번 엑셀 파일을 다시 읽어오는 부하를 방지했습니다. 결측치(NaN)와 에러를 선제적으로 방어한 점이 훌륭합니다.
2. **명확한 모듈화**: `validate_and_load_data`, `preprocess_data`, `render_tab1~4` 등으로 함수를 잘 분리하여 코드가 읽기 쉽고 유지보수가 용이합니다.
3. **사용자 경험(UX) 배려**: 데이터 테이블에 상세 검색(Expander UI 활용)을 넣은 아이디어가 매우 직관적이고 쓸모 있습니다.

### 🤔 비판점 및 개선 필요 사항 (Areas for Improvement)
1. **하드코딩된 절대 경로 (Hardcoded Paths)**: 
   - `c:/dashboard/퐁티_1_300.xls` 및 `C:/Windows/Fonts/malgun.ttf` 같은 절대 경로가 코드에 하드코딩되어 있습니다. 
   - **문제점**: 클라우드(리눅스 환경)나 다른 운영체제(Mac)에 배포할 경우 경로를 찾지 못해 에러가 발생하고 폰트가 깨지거나 워드클라우드 생성이 실패합니다.
2. **단순한 형태소 분석 한계**:
   - 정규식(`[가-힣]{2,}`)을 이용해 2글자 이상의 단어만 무조건 추출하고 있습니다.
   - **문제점**: 동사, 형용사 등 명사가 아닌 엉뚱한 단어나 의미 없는 조사 결합어들이 키워드로 잡힐 확률이 높습니다. 정확한 학술적 키워드를 뽑으려면 한국어 형태소 분석기 도입이 필요합니다.
3. **동적 상호작용성(Interactivity) 부족**:
   - 현재 대시보드는 정적으로 600개의 모든 데이터를 한 번에 보여줍니다.
   - **문제점**: 사용자가 '최근 5년' 혹은 '인용수 10회 이상' 등의 조건으로 데이터를 좁혀서(Filtering) 보고 싶어도 할 수 없습니다. 사이드바(Sidebar) 필터가 부재합니다.

## 2. 대시보드 개선 및 수정 계획 (Improvement Plan)

위 피드백을 바탕으로 코드를 고도화하기 위한 4단계 수정 계획을 제안합니다.

### [Phase 1] 클라우드 완벽 대응: 경로 유연화 및 폰트 내장
- **데이터 경로 수정**: 데이터 파일을 절대경로가 아닌 상대경로(`./퐁티_1_300.xls` 등)로 변경하여 클라우드 구동 시에도 파일을 동적으로 찾을 수 있도록 개선합니다.
- **클라우드용 한글 폰트 적용**: 윈도우 기본 폰트(malgun.ttf)에 의존하는 대신, 클라우드 리눅스 OS에서 폰트 깨짐이 없도록 나눔고딕(NanumGothic) 등 무료 웹 폰트를 프로젝트 내에 포함하거나, OS별로 폰트 경로를 유동적으로 맵핑하는 로직을 추가합니다.

### [Phase 2] 데이터 파이프라인 고도화: KoNLPy 도입
- **형태소 분석기 적용**: 정규식 기반 단순 추출을 폐기하고, 한국어 형태소 분석기 파이썬 패키지인 `KoNLPy` (Okt 등)를 도입하여 정확한 '명사(Noun)' 단위의 개념어만 추출하도록 로직을 재검토합니다.
- **불용어(Stopword) 관리**: 코드 내부에 배열 상태로 있는 불용어를 별도의 `stopwords.txt` 파일로 분리하여 관리의 편의성과 확장성을 높입니다.

### [Phase 3] 대시보드 필터링 시스템 구축 (Sidebar 추가)
- Streamlit의 `st.sidebar`를 도입하여 전역 데이터 필터링 제어판을 구현합니다.
- **추가할 UI 요소**:
  - `st.slider`: 논문 발행연도 범위 선택 필터 (예: 2000년 ~ 2024년)
  - `st.number_input` 또는 `slider`: 최소 인용수 기준 필터
- 이렇게 필터링된 데이터셋(`filtered_df`)을 Tab 1~4의 모든 차트에 동기화하여 동적인 인터랙티브 대시보드를 완성합니다.

### [Phase 4] 의존성 파일(requirements.txt) 구성
- 클라우드 배포 시 인프라가 어떤 패키지를 설치해야 하는지 알 수 있도록 `requirements.txt`를 정확히 작성합니다. (예: `streamlit`, `pandas`, `konlpy`, `plotly`, `wordcloud` 등 포함)
